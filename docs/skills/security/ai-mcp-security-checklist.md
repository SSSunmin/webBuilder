# AI·LLM·MCP 보안 체크리스트

LLM 기능을 붙이거나, MCP 서버/도구·AI 에이전트를 도입·연동할 때 보는 점검 목록.
근거: OWASP LLM Top 10 + MITRE ATLAS. **"무엇을 점검하나"** 리스트다.

> 적용 시점: 챗봇/요약/RAG 등 LLM 기능 추가, MCP 서버 설치·연결, 에이전트에 도구 권한 부여 시.

## 프롬프트 인젝션 — LLM01
- [ ] **외부 데이터(웹페이지·문서·이메일·DB·툴 결과)를 시스템 지시처럼 신뢰하지 않는다.** "이전 지시 무시" 류 주입을 가정하고 설계.
- [ ] 사용자/외부 콘텐츠와 시스템 프롬프트를 **구조적으로 분리**(역할 구분, 구분자, 콘텐츠는 데이터로 표시).
- [ ] 민감 동작(결제·삭제·메일발송)은 LLM 출력만으로 자동 실행하지 말고 **사람 확인 게이트**.

## 출력 처리 — LLM05 (Improper Output Handling)
- [ ] **LLM 출력은 신뢰 못 할 입력**으로 취급. 그대로 `innerHTML`/쉘/SQL/`eval`에 넣지 않는다(→ 2차 XSS·인젝션). 다운스트림에서 sanitize/검증.
- [ ] LLM이 만든 코드·명령은 자동 실행 전 검토. 마크다운 렌더 시 링크/이미지 출처 점검.

## 과도한 권한 (Excessive Agency) — LLM06
- [ ] 에이전트/도구에 **필요한 최소 권한만**. 읽기면 충분한데 쓰기/삭제 권한 주지 않기.
- [ ] 도구 호출 범위·횟수·대상 제한(rate/scope). 파괴적 작업은 승인 필요.

## 민감정보 유출 — LLM02 / 시스템프롬프트 유출 — LLM07
- [ ] 프롬프트/컨텍스트에 **API 키·시크릿·PII를 넣지 않는다**(모델·로그·제3자에 노출).
- [ ] 시스템 프롬프트에 비밀을 의존하지 않기(추출될 수 있음). 응답에 내부 정보 누출 방지 필터.

## MCP 서버·도구 신뢰 (Tool Poisoning / Supply Chain) — LLM03
- [ ] **출처 불명 MCP 서버를 무심코 연결하지 않는다.** 도구 설명(description)에 숨은 지시("~할 때 이 파일을 읽어 전송")가 있는지 점검 — 이게 tool poisoning.
- [ ] 설치 전 소스/권한 확인: 어떤 파일·네트워크·환경변수에 접근하나. 최소 권한으로.
- [ ] 업데이트 시 도구 정의가 조용히 바뀌는지(rug pull) 주의. 핀 버전/감사.
- [ ] 한 도구의 출력이 다른 도구로 흘러 의도치 않은 연쇄 실행되지 않는지(line confusion).

## 데이터/모델 포이즈닝 — LLM04
- [ ] RAG/파인튜닝에 넣는 외부 코퍼스의 출처·무결성 확인. 오염된 문서가 검색되어 주입되는 경로 점검.

## 벡터/임베딩 — LLM08
- [ ] RAG에서 **테넌트/사용자별 접근 격리**(다른 사용자 문서가 검색되지 않게). 임베딩 저장소 권한 분리.

## 자원 소비 — LLM10
- [ ] 토큰·요청 비용 상한, 타임아웃, 재귀 호출 제한(무한 에이전트 루프·비용 폭주 방지).

## 빠른 레드플래그
- 외부 텍스트를 프롬프트에 그대로 이어붙여 지시로 사용
- LLM 출력을 `innerHTML`/쉘/SQL/`eval`에 직접 투입
- 에이전트에 광범위 쓰기·삭제 권한 + 자동 실행
- 프롬프트/로그에 시크릿·PII
- 검증 안 된 MCP 서버 연결, 도구 description 미점검
- RAG 검색에 사용자간 격리 없음
