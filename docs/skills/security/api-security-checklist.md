# 백엔드/API 보안 체크리스트

백엔드·API 코드를 작성/리뷰할 때 보는 점검 목록. 언어/프레임워크 무관(Node·Python·Java·Go 등 공통).
근거: OWASP API Security Top 10 (2023) + OWASP Top 10. **"무엇을 점검하나"** 리스트다.

> 적용 시점: 엔드포인트 추가/수정, 인증·권한, DB 쿼리, 외부 호출, 파일·업로드를 다룰 때.

## 권한 (가장 흔한 실제 취약점)
- [ ] **객체 수준 권한(BOLA/IDOR)**: `/orders/{id}` 같은 리소스 접근 시 "이 사용자가 이 객체의 주인인가"를 **매 요청 서버에서 검증**. ID를 받았다고 곧바로 조회 금지. (API1)
- [ ] **함수 수준 권한(BFLA)**: 관리자 전용 엔드포인트를 일반 사용자가 못 부르게 역할 검사. URL 추측으로 접근 안 되는지. (API5)
- [ ] **속성 수준**: 응답에서 민감 필드(비밀번호 해시·내부 플래그) 제외, 요청에서 권한필드(`isAdmin`, `role`) **mass assignment** 차단(허용 필드 화이트리스트). (API3)

## 인증
- [ ] 강한 인증(토큰 서명·만료 검증). **JWT `alg:none` / 약한 시크릿** 거부, 서명 검증 필수. (API2)
- [ ] 무차별 대입 방어: 로그인·OTP·비번재설정에 rate limit + 계정 잠금.
- [ ] 비밀번호 저장은 **bcrypt/argon2** 등 느린 해시. 평문·MD5·SHA1 금지.

## 입력 검증·인젝션 — A03
- [ ] **신뢰 경계의 모든 입력 검증**(타입·범위·길이·형식). 화이트리스트 우선.
- [ ] SQL/NoSQL: **파라미터 바인딩/프리페어드 스테이트먼트**만. 쿼리 문자열 연결 금지.
- [ ] OS 명령·경로: 사용자 입력으로 `exec`/파일경로 조합 금지(커맨드 인젝션·경로 탐색). 경로는 정규화 후 화이트리스트.
- [ ] 역직렬화: 신뢰 못 할 데이터를 임의 객체로 역직렬화 금지.

## SSRF — API7 / A10
- [ ] 사용자가 준 URL로 서버가 요청을 보낼 때(웹훅·이미지 프록시·미리보기) **내부망/메타데이터(169.254.169.254)/localhost 차단**. 출력 IP 화이트리스트, 리다이렉트 추적 제한.

## 자원 소비 — API4
- [ ] **rate limit / 페이지네이션 상한 / 페이로드 크기 제한**. 무제한 목록·대용량 업로드·재귀 쿼리 방지.
- [ ] GraphQL: 쿼리 깊이·복잡도 제한.

## 설정·노출 — API8 / A05
- [ ] **에러 응답에 스택트레이스·내부경로·SQL 노출 금지**. 사용자에겐 일반 메시지, 상세는 서버 로그.
- [ ] CORS 출처 화이트리스트(쿠키 인증 시 `*` 금지). 불필요한 HTTP 메서드/디버그 엔드포인트 비활성.
- [ ] 보안 헤더: HTTPS 강제(HSTS), 적절한 캐시 정책.
- [ ] 시크릿은 환경변수/시크릿매니저. 코드·레포·로그에 하드코딩 금지.

## 로깅·모니터링 — A09
- [ ] 인증 실패·권한 거부·이상 행위 로깅. 단 **로그에 비밀번호·토큰·PII 평문 금지**.

## 외부 API 신뢰 — API10
- [ ] 서드파티 API 응답도 검증 후 사용(맹신 금지). TLS 검증 끄지 않기.

## 빠른 레드플래그
- 쿼리 문자열 연결(`"... WHERE id=" + req`) / 사용자 입력으로 `exec`
- 객체 ID만 받고 소유자 확인 없이 조회/수정
- 요청 바디를 그대로 모델에 `save`(mass assignment)
- 에러를 그대로 클라이언트에 반환
- 사용자 URL로 서버가 fetch(SSRF) / TLS 검증 비활성
- 시크릿 하드코딩 / 로그에 토큰
