# 피그마 스킬팩 (조건부)

피그마 디자인을 코드로 정교하게 옮기기 위한 **온디맨드 문서 묶음**.
**범용 규칙만** 담았다(특정 디자인 시스템 값 X). 어느 피그마 프로젝트든 바로 쓸 수 있다.

> ⚠️ **조건부 — 이 폴더는 피그마를 쓰는 프로젝트에만 둔다.**
> - 피그마를 안 쓰면 `docs/skills/ui/figma/` **폴더 통째로 삭제**(자기완결형이라 안전).
> - **`figma-shadcn.md`는 shadcn/ui를 쓰는 프로젝트에만** 남긴다. shadcn 안 쓰면 그 파일만 삭제.

## 문서 (작업별로 필요한 것만 읽기)

| 작업 | 문서 | 조건 |
|---|---|---|
| 피그마에서 읽기·추출 (MCP·node-id·대용량·asset 색상) | `figma-mcp.md` | 범용 |
| 정확히 옮기기 (색상 추측금지·모든 속성·토큰우선·레이아웃·완료 전 대조) | `figma-styling.md` | 범용 |
| 아이콘 SVG 추출·등록 (fill/stroke·viewBox 크기) | `figma-icons.md` | 범용 |
| shadcn 컴포넌트 디자인 맞추기 (기본값 재정의) | `figma-shadcn.md` | **shadcn 쓸 때만** |

## 프로젝트 특화값은 여기 적지 않는다 → 불변규칙으로
토큰 명명 케이스(`bg-primary-light` vs `bg-Primary-Light`), 단위 환산(예: `1rem = N px`), 토큰/아이콘/이미지 경로,
모달·콘텐츠 너비 수치 등은 **프로젝트마다 다르다.** 이런 건 이 문서에 박지 말고,
`/setup-rules`로 루트 `CLAUDE.md`의 **절대 불변규칙**에 적는다(또는 그 프로젝트 전용 skill로). 이 문서들의 예시 수치는 *형식 예시일 뿐*이다.

## 도입 시
1. 이 폴더를 피그마 프로젝트에 둔다(안 쓰면 삭제, shadcn 안 쓰면 `figma-shadcn.md` 삭제).
2. 루트 `CLAUDE.md`의 "온디맨드 라우터" 표에 `UI·피그마 → docs/skills/ui/figma/*.md` 한 줄 등록.
3. `/setup-rules`로 토큰 명명·단위·경로 같은 **프로젝트 피그마 불변규칙**을 채운다.
