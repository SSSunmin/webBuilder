# UltraQA — 자율 QA 사이클

목표가 충족될 때까지 도는 검증 루프다. **사이클: 검증 → 진단 → 수정 → 반복 (최대 5회)**.

## 목표 파싱 ($ARGUMENTS)
| 인자 | 무엇을 확인 |
|---|---|
| `--tests` | 프로젝트 테스트 명령이 모두 통과 |
| `--build` | 빌드가 exit 0 |
| `--lint` | 린트 에러 없음 |
| `--typecheck` | 타입 에러 없음 |
| `--custom "패턴"` | 적절한 명령 실행 후 출력에 패턴 존재 |
| `--interactive` | `qa-tester`로 앱을 띄워 인터랙티브 검증 |
인자가 없으면 사용자에게 목표를 1줄로 확인하고 시작한다. 명령(테스트/빌드 등)은 `package.json`/설정에서 **실제** 스크립트를 읽어 쓴다(추측 금지).

## 사이클 (각 회차)
1. **검증 실행**: 목표 유형의 명령을 Bash로 실행(`--interactive`면 `qa-tester` 에이전트에 위임).
2. **결과 판정**: 통과면 → 성공 종료. 실패면 → 3으로.
3. **진단**: `debugger` 에이전트(없으면 `architect`)에 실패 출력을 주고 **근본원인 + 구체적 수정안**을 받는다.
4. **수정**: 진단대로 고친다. 한두 파일·소규모면 직접, 멀티파일이면 서브에이전트에 위임. **수정한 그 패스에서 통과를 자가선언하지 않는다** — 반드시 1로 돌아가 재검증.
5. **반복**.

## 종료 조건
| 조건 | 동작 |
|---|---|
| 목표 달성 | `ULTRAQA 완료: N회만에 목표 달성` |
| 5회 도달 | `ULTRAQA 중단: 최대 회차. 진단: …` |
| **같은 실패 3회** | 조기중단 `ULTRAQA 중단: 동일 실패 3회. 근본원인: …` |
| 환경 오류 | `ULTRAQA 오류: [포트/의존성/도구 문제]` |

## 진행 출력 (매 회차)
```
[UltraQA 1/5] 테스트 실행 중…
[UltraQA 1/5] 실패 — 3건 실패 (auth.test.ts)
[UltraQA 1/5] 진단 중…
[UltraQA 1/5] 수정: auth.test.ts — mock 누락
[UltraQA 2/5] 테스트 실행 중…
[UltraQA 2/5] 통과 — 47건 전부 통과
[UltraQA 완료] 2회만에 달성
```

## 규칙
- **실패 패턴 추적**: 회차별 실패를 기록해 동일 실패 3회면 조기중단(무한루프 방지).
- **증거 우선**: 매 회차 실제 명령 출력으로 통과/실패를 판정한다. 추측으로 "통과" 금지.
- **작성·검증 분리**: 수정한 패스에서 스스로 OK 하지 말고 다음 회차 재검증으로만 확정.
- 이 커맨드는 OMC 의존이 없다. 진단은 키트의 `debugger`/`architect`, 인터랙티브 검증은 `qa-tester`를 쓴다.
