description: 최근 변경 코드를 동작 보존하며 더 단순·일관되게 다듬는다 (리뷰 다음, 검증 직전)
---
방금 변경한 코드를 **동작은 그대로 두고** 더 단순·명확·일관되게 다듬어줘.

- 기능·공개 동작·에러 처리를 **100% 보존**한다. 동작이 바뀔 것 같으면 고치지 말고 보고한다.
- 중복 제거·네이밍·죽은 코드 정리·깊은 중첩 완화·일관성에 집중한다.
- **주변 코드 관례를 따른다** — 내 취향으로 통일하지 않는다. 과도한 추상화 금지.
- 최근 변경분에만 집중한다(별도 지시 없으면 전체 리팩터 금지).
- 다듬은 뒤 가능하면 타입체크/린트/테스트로 동작 불변을 확인하고 결과를 보여준다.

규모가 크면 `code-simplifier` 에이전트에 위임해도 좋다.
순서상 `/review`(지적) 다음, `/verify`·`/ultraqa`(검증) 직전에 쓴다.
