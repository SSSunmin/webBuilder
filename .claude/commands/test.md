---
description: 최근 변경된 코드에 테스트를 작성하고 실행해 통과를 확인한다
argument-hint: "[파일/영역 (생략 시 git diff 기준)]"
---
방금 변경한 코드(`$ARGUMENTS` 지정 시 그 범위, 없으면 `git diff` 기준)에 대해 테스트를 작성하고 실행해줘. `test-writer` 에이전트에 위임한다.

- **기존 테스트 프레임워크·관례를 먼저 확인**하고 그대로 따른다(새 라이브러리 임의 도입 금지).
- 로직(함수·훅·리듀서·검증·변환 등)은 **행복경로 + 엣지 + 에러 케이스**를 덮는다.
- **동작 보존이 목표인 변경(리팩터·리네임·이동)**은 변경 전 현재 동작을 핀하는 **특성화 테스트를 먼저** 작성해, 변경 후에도 통과하는지로 동작 보존을 증명한다.
- 순수 스타일/마크업처럼 단위 테스트가 무의미한 변경은 **작성하지 말고 이유를 보고**한다(가짜 커버리지 금지).
- **가짜 테스트 방어**: 작성한 테스트가 *틀린 구현에서 실패하는지* 먼저 확인한다(see-it-fail). 핵심 로직(돈·권한·데이터·신뢰경계 — 판정·뮤테이션 기준은 `docs/skills/foundation/test-quality.md`)엔 강한 검증을 집중.
- **작성 후 실제로 실행**해 통과를 증명한다. Playwright는 반드시 Bash 툴로 실행.
- 테스트가 **제품 코드의 진짜 버그**를 드러내면 코드를 고치지 말고 보고한다(작성·리뷰 분리).
